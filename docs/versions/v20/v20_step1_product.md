# V20: 运行中上下文插入能力 - 产品文档

## 目标

让人类与 Agent 的私聊协作更加顺手：当 Agent 正在工作时，人类不必总是等它整轮结束后，才能补充新的上下文或纠正方向。

此前系统中的新消息通常要等当前 turn 结束后，才会在下一轮进入 Agent 的处理流程。这在私聊场景下会带来明显延迟：人类已经发现 Agent 方向偏了，或者想临时补充信息，但只能先等它把当前整轮走完。V20 之后，在私聊房间中发送的新消息可以选择“尽快插入当前工作过程”，让 Agent 在不中断当前工具执行的前提下，更早感知到新的上下文。

---

## 功能特性

### 一、私聊补充信息可更快生效

- **运行中补充**：当 Agent 正在私聊房间中工作时，人类可以继续发送新的补充信息，而不必只能等待整轮结束。
- **尽快影响后续行为**：新消息发出后，会尽快进入 Agent 当前工作过程，对后续判断、回复或工具选择产生影响。
- **减少等待感**：人类不需要频繁经历“我已经补充了信息，但还要等很久才被看到”的体验。

### 二、不粗暴打断当前动作

- **当前步骤先完成**：如果 Agent 正在执行某个工具动作，新消息不会直接把当前动作掐断。
- **在自然边界衔接**：系统会在当前步骤结束后的合适时机，让新消息参与后续处理。
- **降低混乱感**：既能更快纠偏，又不会因为半路强行插入而让 Agent 的行为变得不可预测。

### 三、只用于私聊纠偏场景

- **私聊优先支持**：该能力只面向人类与单个 Agent 的私聊房间开放。
- **群聊保持稳定**：多人群聊房间继续沿用原有消息节奏，不引入运行中插入，避免打乱群体协作和轮次理解。
- **适合一对一监督**：这个版本聚焦“人类盯着一个 Agent 干活，并及时补充信息”的典型产品场景。

### 四、消息发送方式保持简单

- **仍然是发送房间消息**：用户不需要学习新的操作模型，仍然是在私聊房间发送消息。
- **增加一个可选语义**：发送时可指定“这条消息希望尽快影响当前过程”，而不是只能作为普通后续消息排队。

### 五、范围先收敛到主流运行模式

- **先覆盖当前主路径**：本版本先确保当前主流的 host-managed Agent 工作流中，私聊补充信息可以工作正常。
- **不追求一步到位**：其他运行模式暂不纳入本版本范围，先把最常见、最有价值的产品路径做顺。

---

## 用户价值

### 1. 纠偏更及时

当人类发现 Agent 理解错了任务、缺少关键信息、或即将走向错误方向时，可以更快补一句说明，而不是只能等它整轮结束。

### 2. 协作更自然

私聊协作会更像真实对话：人类可以边看边补充，Agent 也能更早吸收这些变化，而不是一轮一轮地机械等待。

---

## 核心概念

### 运行中插入

指 Agent 当前正在处理一次私聊任务时，新发送的私聊消息可以在当前工作过程尚未完全结束前，被后续处理流程吸收，而不必只能等到下一整轮。

### 私聊优先

该能力只针对人类与单个 Agent 的私聊协作场景设计，不改变多人群聊的协作模式。

### 自然边界生效

系统不会粗暴打断 Agent 正在做的当前动作，而是在当前步骤完成后的自然衔接点，让新消息参与后续处理。

---

## 效果演示

### V20 之前

```text
Operator：先检查 stderr
Agent：正在执行 read_file / execute_bash / 推理……

结果：
- Operator 的新消息已经发出
- 但 Agent 要等当前整轮结束后，下一轮才会看到
```

### V20 之后

```text
Operator：先检查 stderr
Agent：正在执行 execute_bash

结果：
- 当前 execute_bash 先执行完
- Agent 在后续推理前看到“先检查 stderr”这条新消息
- 后续行为尽快转向新的要求
```

### 群聊场景保持不变

```text
房间：产品组讨论室（多人群聊）
Operator：补充一句说明

结果：
- 消息仍按原有群聊节奏流转
- 不引入运行中插入能力
```

---

## 产品边界

### V20 包含

- 私聊房间中的新消息可选择尽快影响 Agent 当前工作过程
- Agent 正在执行工具时，新消息在当前步骤结束后参与后续处理
- 该能力仅支持私聊房间
- 先支持当前主流的 host-managed 工作流

### V20 不包含

- 多人群聊房间的运行中插入
- 面向所有 driver 的完整统一支持
- 人工强制中断当前动作的能力

---

## 验收标准

### 私聊效果

- 在私聊房间中，向正在工作的 Agent 发送一条“尽快生效”的消息后，该消息可在当前工作过程内影响后续行为
- 用户不必等待整个 turn 全部结束，便能看到新的私聊消息开始产生效果

### 工具执行边界

- 当 Agent 正在执行工具动作时，新消息不会导致当前工具调用直接中断
- 当前步骤完成后，Agent 的后续处理能体现这条新消息的影响

### 范围控制

- 多人群聊房间中不支持该能力，行为与之前保持一致
- 非主流运行模式不要求在本版本支持

### 使用体验

- 用户仍通过发送私聊消息完成操作，无需学习新的“隐藏指令通道”
- 功能语义清晰：普通私聊消息继续可用，“尽快生效”的消息表现符合预期
